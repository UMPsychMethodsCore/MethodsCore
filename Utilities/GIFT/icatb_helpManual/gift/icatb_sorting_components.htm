Sorting Components

Sorting is a way to classify the components. For every independent component spatial maps and time courses are generated. Components can be sorted in two ways:

  • Temporal Sorting
  • Spatial Sorting

Select 'Yes' for sort components in the Component Explorer and click Display button then a dialog box will pop up that shows the information about the sorting. Click Ok for the dialog button and then a figure will open in a new window. Figure below shows the completed parameters.

Figure 1: Sorting GUI

The following are the definitions of the parameters:

Sorting Criteria
There are three different types of sorting criteria like Multiple Regression, Correlation, Kurtosis and Maximum Voxel.
What do you want to sort?
There are four options for temporal sorting depending upon the number of subjects and sessions you select.
  • Select any Subject any session - A particular subject's session is selected and correlated with the model time course.
  • Select a particular session of all subjects - A particular session for all subjects is selected, time courses are concatenated and correlated with the concatenated model time course.
  • Select a particular Subject - A particular subject's sessions are selected, time courses are concatenated and correlated with the model time course.
  • Select all Subjects and Sessions - All data sets are concatenated and correlated with the model time course.
Sorting Type
Temporal or Spatial.
Reference Function
You have a good measure to separate the task related components depending on the model time course or spatial template.
